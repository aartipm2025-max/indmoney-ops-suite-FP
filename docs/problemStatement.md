# INDmoney Investor Ops & Intelligence Suite — Problem Statement

## Users (Personas)

### Persona 1: Priya, Ops Analyst at INDmoney
Priya processes inbound customer reviews and escalations across mutual fund, stock, and fixed-income products. Every week she receives approximately 10,000 reviews from app stores, support tickets, and NPS surveys. Her current workflow is entirely manual: she reads a random sample, writes a summary in a Google Doc, and forwards it to the product and advisory teams.

**Core pains:**
- No automated triage: urgent complaints (redemption failures, KYC blocks) surface days late.
- No trend detection: she cannot tell whether a spike in "SIP failure" complaints is a new pattern or seasonal noise.
- Manual weekly summary takes 3–4 hours every Monday; it is subjective and inconsistent week to week.
- No citation trail: when PMs ask "where did you see that?", she cannot quickly reconstruct the source reviews.

### Persona 2: Raj, Financial Advisor at INDmoney
Raj handles high-value investor accounts and conducts 8–12 advisory calls per day. Before each call he has no structured briefing: he checks the account screen, reads a few recent transactions, and improvises.

**Core pains:**
- Walks into calls blind: no automated summary of what the customer has asked about recently, or what fund topics are generating complaints across similar customer segments.
- No theme awareness: cannot anticipate that "exit load confusion" is the top theme this week among debt-fund investors — so he gets blindsided mid-call.
- Post-call admin is manual: he writes follow-up emails and calendar holds by hand, 10–15 minutes per call.

## Primary Use Cases

### UC1: Unified Knowledge Base Lookup (Pillar A — RAG)
An ops analyst or advisor types a natural-language question such as "What is the exit load for HDFC Flexicap if I redeem within 18 months, and what does the factsheet say about the fund's expense ratio?"

The system retrieves chunks from fund factsheets and fee-schedule documents simultaneously (hybrid BM25 + vector search), reranks them, and returns a 6-bullet cited answer where every bullet includes a `[source:doc_id]` tag. No answer is given without a citation; no investment advice is generated.

### UC2: Weekly Review-to-Pulse Workflow (Pillar B — Pulse)
Priya uploads a CSV of 300 customer reviews (2-week window). The system:
1. Detects top 3 complaint themes using clustering + LLM labelling.
2. Identifies whether each theme is rising, stable, or falling vs. the prior 2-week window.
3. Generates a Weekly Pulse: ≤250-word summary, exactly 3 actionable recommendations, each anchored to specific review clusters.
4. Completes in under 60 seconds end-to-end.

### UC3: Theme-Aware Voice Call Briefing with HITL Follow-up (Pillar C — Voice + HITL)
Before Raj's next call, the system:
1. Reads the customer's recent interaction history + current top themes.
2. Generates a spoken briefing opening that names the #1 theme relevant to that customer segment.
3. After the call, generates a Gmail draft (real OAuth) and a Google Calendar hold (real OAuth).
4. Both artifacts go through a Human-in-the-Loop approval step: Raj reviews, edits if needed, and approves. No email is sent or calendar event created without explicit approval.

| Gmail draft | Real Gmail API draft | HITL required before send |
| Calendar hold | Real Google Calendar event | HITL required before create |
| Eval report | Markdown + JSONL | Auto-generated by `evals/run_evals.py` |

---

## Constraints

- **No PII stored**: all customer names in review CSVs are replaced with `[REDACTED]` before any LLM call or database write.
- **No investment advice**: every LLM prompt includes a hard guardrail; any response that constitutes investment advice triggers `SafetyViolationError` and returns a refusal message. SEBI compliance is non-negotiable.
- **HITL required**: no customer-facing email or calendar event is created without an explicit human approval step in the Streamlit UI.
- **100% free stack**: Groq free tier (LLM), ChromaDB (local vector store), SQLite (state), Google APIs (OAuth free tier), Streamlit Community Cloud (hosting). No paid APIs.
- **9-day build, solo developer**: scope is strictly phase-gated. Nothing is built speculatively.
